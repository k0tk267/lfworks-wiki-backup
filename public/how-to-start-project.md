# プロジェクトの進め方

「イシューからはじめよ」を聖書とし、この方針に則ってプロジェクトを進める。
人間はAIと並走し、課題創出とラストワンマイルの作業のみに集中する。

1. [Project管理ページ](https://github.com/orgs/lfworks/projects)にプロジェクトを作成する
2. Issueを切る
3. やる
4. 課題のステータスをdoneに持っていく

という流れ。
兎に角、有益なIssueを書くことに頭のリソースを最大限使う。
無駄なことは頑張っても無駄（犬の道）。

## 「やる」について
コードの実装が必要な話であれば、基本的にclaudeに任せて人間はレビューに徹する。
Issueを適切に書けば、@claudeでclaudeが自動でPRまで作成してくれるように設定した。
（ただしIssueを該当リポジトリにtransferする必要がある）

とにかく「なにをやるか」と「どうやるか」を言語化してまとめる方向に時間を割くことにする。


## 怠惰
できるだけすべて自動化できるなら自動化する方向で進めたい。
面倒な作業はできるだけしない。
どれだけ創造性の高い部分に人間のリソースを突っ込めるか考える方向で進める。

とはいえ、自動化の前に人間がパターンを学習し、言語に落とす過程で生じる行動は重要とする。
